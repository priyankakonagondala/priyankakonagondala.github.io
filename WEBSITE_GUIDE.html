    How Quarto Works

    Quarto is a publishing system that converts .qmd (Quarto Markdown) files into HTML pages. Think of it like this:

    1. You write content in .qmd files (plain text with optional front matter)
    2. Quarto reads _quarto.yml for site-wide settings (theme, navbar, sidebar)
    3. Quarto renders each .qmd into an .html file in _site/
    4. You deploy the _site/ folder (to GitHub Pages, Netlify, etc.)

    .qmd File Anatomy

    Every .qmd file has two parts:

    ---
title: "My Page Title"
date: 2026-07-07
---

This is the body content, written in **Markdown**.
    • Front matter (between ---): YAML metadata (title, date, categories)
    • Body: Markdown content (headings, paragraphs, links, images)

    Where to Change Things

    1. Colors and Theme → styles.css

    All visual design lives here. Key sections:

    What to change Where in styles.css
    Background color (#00072D) body { background-color: #00072D; }
    Navbar color (#011040) .navbar { background-color: #011040 !important; }
    Link color (#5dade2) a { color: #5dade2; }
    Text color color: #d4d4d4 on body, color: #ccc on paragraphs
    Sidebar styling #quarto-sidebar { ... }
    Footer .page-footer { ... }

    To change the entire color scheme, search-and-replace the hex codes: - #00072D → your main background color - #011040 → your navbar/sidebar color - #0a2463 → your border/divider color - #5dade2 → your link/accent color

    2. Navigation Bar → _quarto.yml

    navbar:
  left:
    - text: "Home"      # What appears in the nav
      href: index.qmd   # Which page it links to

    To add a new nav item, add a - text: "New Section" entry.

    3. Book Sidebar → _quarto.yml

    sidebar:
  contents:
    - section: "The Limits of Scale"
      collapse: true           # Makes it collapsible
      contents:                # All chapters go here
        - my-book/ch1.qmd
        - my-book/ch2.qmd

    When you add a new chapter, add its .qmd file to this list.

    4. Home Page Content → index.qmd

    Edit the bio text, links, and sections directly in this file.

title: "Chapter 1 Title"

---

```{=html}
<script>document.body.classList.add('book-page');</script>

    Your chapter content here…

    
The `<script>` line is important — it tells the site to show the sidebar only on book pages.

    Step 4: Add a book entry card in posts/

    How to Deploy

    GitHub Pages

    1. Push this repo to GitHub as username/username.github.io
    2. Go to Settings → Pages → Source → Deploy from branch → main
    3. Your site will be live at https://username.github.io

    Manual Deploy

    1. Run quarto render
    2. Upload the _site/ folder contents to any web host (Netlify, Vercel, S3, etc.)

    Common Troubleshooting

    Problem Fix
    Styles not updating Delete _site/ and .quarto/ folders, then quarto render
    Preview server errors pkill -f "quarto preview" then restart
    Sidebar showing on all pages Check that book chapters have the book-page script
    Blog post not in listing Make sure the .qmd is in posts/ and has a title: and date:
